桥接模式（Bridge Pattern）

桥接模式是一种结构型设计模式，它的核心思想是将抽象部分与实现部分分离，使它们可以独立地变化。这种模式通过建立抽象和实现之间的桥梁，让两者可以在不互相影响的情况下扩展。

核心概念

• 抽象（Abstraction）：定义抽象类的接口，维护一个对实现类的引用。

• 扩充抽象（Refined Abstraction）：扩展抽象类的接口。

• 实现接口（Implementor）：定义实现类的接口，与抽象类的接口可能不同。

• 具体实现（Concrete Implementor）：实现实现接口的具体类。

为什么需要桥接模式？

当一个类存在两个独立变化的维度（如抽象和实现），且需要分别扩展时，桥接模式可以避免使用多层继承导致的类爆炸问题。

案例示例：手机软件与硬件的桥接

假设我们要设计手机系统，手机品牌（硬件）和手机功能（软件）是两个独立变化的维度。使用桥接模式可以让它们独立扩展。
// 实现接口：手机品牌（硬件维度）
interface PhoneBrand {
void open();
void close();
void call();
}

// 具体实现：不同品牌的手机
class PhoneBrandA implements PhoneBrand {
@Override
public void open() {
System.out.println(“品牌A手机开机”);
}
@Override
public void close() {
System.out.println(“品牌A手机关机”);
}
@Override
public void call() {
System.out.println(“品牌A手机打电话”);
}
}

class PhoneBrandB implements PhoneBrand {
@Override
public void open() {
System.out.println(“品牌B手机开机”);
}
@Override
public void close() {
System.out.println(“品牌B手机关机”);
}
@Override
public void call() {
System.out.println(“品牌B手机打电话”);
}
}

// 抽象类：手机功能（软件维度）
abstract class PhoneFunction {
protected PhoneBrand brand;

public PhoneFunction(PhoneBrand brand) {
    this.brand = brand;
}

public abstract void run();

}

// 扩充抽象：具体手机功能
class PhoneGame extends PhoneFunction {
public PhoneGame(PhoneBrand brand) {
super(brand);
}

@Override
public void run() {
    brand.open();
    System.out.println("运行手机游戏");
    brand.call(); // 游戏中可能需要通话功能
    brand.close();
}

}

class PhoneAddressList extends PhoneFunction {
public PhoneAddressList(PhoneBrand brand) {
super(brand);
}

@Override
public void run() {
    brand.open();
    System.out.println("运行通讯录功能");
    brand.close();
}

}

// 客户端测试
public class BridgePatternDemo {
public static void main(String[] args) {
// 品牌A手机运行游戏功能
PhoneFunction gameOnBrandA = new PhoneGame(new PhoneBrandA());
gameOnBrandA.run();

    // 品牌B手机运行通讯录功能
    PhoneFunction addressListOnBrandB = new PhoneAddressList(new PhoneBrandB());
    addressListOnBrandB.run();
}

}
桥接模式的结构
抽象类(Abstraction)
├─ 扩充抽象类(RefinedAbstraction)
│
实现接口(Implementor)
├─ 具体实现类(ConcreteImplementorA)
└─ 具体实现类(ConcreteImplementorB)
桥接模式的优点

  1. 分离抽象和实现：使两者可以独立扩展，符合开闭原则。

  2. 减少子类数量：避免多层继承导致的类爆炸。

  3. 提高可扩展性：抽象和实现的扩展互不影响。

  4. 符合单一职责原则：抽象层和实现层各负其责。

应用场景

• 抽象和实现需要独立变化：如跨平台软件（抽象功能与具体平台实现分离）。

• 避免多层继承：当系统有多个维度变化时，桥接模式比多层继承更灵活。

• 框架设计：如Java的AWT（抽象窗口工具包）中，将抽象组件与具体操作系统实现分离。

• JDBC驱动程序：将数据库操作抽象与具体数据库实现分离。

与其他模式的区别

• 与适配器模式：适配器模式用于转换接口，桥接模式用于分离抽象和实现。

• 与策略模式：策略模式封装算法族，桥接模式处理抽象和实现的关系。

通过桥接模式，我们可以将系统中的多个变化维度解耦，使代码结构更清晰，扩展更灵活。
